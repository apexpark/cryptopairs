# Operations Governance

This folder contains operator-facing workflow guidance for agentic development in
this repository. It does not replace `AGENTS.md`; if anything conflicts,
`AGENTS.md` wins.

## Documents

- `docs/ops/ai_workflow.md` - Apex harness methodology for Operator, Coder,
  and Reviewer collaboration.
- `docs/ops/codex_prompt_pack.md` - reusable prompts for Coder startup,
  Reviewer handoff, Operator signoff, and exception requests.
- `docs/research/packets/template.md` - template for future governance or
  research packets.
- `docs/research/packets/01-agentic-harness.md` - first packet defining the
  harness decision and repository-specific lanes.

## Operating Rule

Use these documents for repository governance and review discipline. Use the
domain policies in `docs/10-16` for technical safety decisions, and the
playbooks in `docs/playbooks/` for operational procedures.
